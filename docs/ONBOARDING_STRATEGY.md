# NT-ClawLaunch: 安裝導引 (Onboarding) 核心策略

對於封裝底層技術的開源工具來說，**安裝導引（Onboarding / Setup Wizard）是決定用戶去留的「生死線」**。

## 🎨 4 大核心 UI/UX 設計秘訣

### 1. 視覺化的「步驟條」與目標拆解 (Stepper UI)
* **設計邏輯**：避免單一長表單。流程應切分為：`環境檢測 -> 選擇模型與 API -> 連接通訊軟體 -> 選擇 Skill -> 啟動`。
* **UI 表現**：頂部明確顯示「Step 1 of 5」，減少使用者焦慮。

### 2. 情感化設計與「偽裝」的等待時間 (Loading States)
* **設計邏輯**：背景安裝時（Node.js/Git），需以動畫安撫使用者。
* **UI 表現**：使用「蝦爪跳動」動畫，並配上人性化文案（如：「正在為您佈置溫馨的蝦窩...」）。

### 3. 「保姆級」的無縫跳轉與內聯幫助 (Frictionless Help)
* **獲取 API Key 的捷徑**：輸入框旁附帶直接跳轉連結。
* **圖文教程彈窗**：針對複雜操作（如飛書配對），直接提供帶截圖的內嵌教程。

### 4. 專業術語的「白話文翻譯」 (Jargon Translation)
* **設計邏輯**：將技術名詞（Hook, Browsers-Skill）轉化為白話（控制 Mac UI, 蘋果備忘錄聯動）。
* **UI 表現**：保持介面乾淨，深入解釋僅在「懸浮提示 (Tooltip)」中顯示。

## 🛠️ 實作路線圖 (Phase 500+)
1. **實作核心 Stepper 元件**：建立可切換的步驟容器。
2. **開發「注入靈魂」模型選擇器**：參考 Neil 提供之 React + Tailwind 代碼。
3. **整合環境自動檢測**：於第一步自動檢查 Git/Node/Python 狀態。
4. **品牌動畫注入**：於 Electron 載入資源時顯示蝦爪動畫。
