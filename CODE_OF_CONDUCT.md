# Code of Conduct

## Our Pledge

As members, contributors, and maintainers of the NT-ClawLaunch project, we pledge to provide a harassment-free environment for everyone, regardless of background, gender identity, sexual orientation, disability, physical appearance, body size, race, age, religion, or technical level.

We are committed to maintaining a friendly, safe, and inclusive community.

## Our Standards

To foster a positive environment, we encourage:

- Demonstrating empathy and kindness toward others
- Respecting differing viewpoints and experiences
- Giving and gracefully accepting constructive feedback
- Taking responsibility and apologizing when we make mistakes
- Focusing on what is best for the overall community

Unacceptable behavior includes:

- Using sexualized language or imagery
- Personal attacks, bullying, humiliation, or denigration
- Publishing others' private information without explicit permission
- Harassment in any form, public or private
- Other conduct that could reasonably be considered inappropriate in a professional setting

## Maintainer Responsibilities

Project maintainers are responsible for clarifying the standards of this Code of Conduct and will take appropriate and fair corrective action in response to any behavior they deem inappropriate, threatening, offensive, or harmful.

## Scope

This Code of Conduct applies to all project-related spaces, both virtual and in-person, including GitHub Issues, Pull Requests, discussions, and social media.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please contact the project owner or a maintainer (for example, through the project's development channel). All reports will be handled confidentially.

# 行為準則

## 我們的承諾

身為 NT-ClawLaunch 專案的成員、貢獻者與維護者，我們承諾為每個人提供一個無騷擾的環境，不論其背景、性別認同、性取向、殘疾、生理外觀、體型、種族、年齡、宗教或技術層次。

我們致力於維護一個友善、安全且具備包容性的社群。

## 我們的標準

為了營造正向環境，我們鼓勵以下行為：

- 展現同理心並善待他人
- 尊重不同的觀點與經驗
- 提供並優雅接受建設性的反饋
- 承擔責任，並在犯錯時道歉
- 專注於對整體社群最有利的事情

不可接受的行為包括：

- 使用性暗示語言或意象
- 進行人身攻擊、霸凌、羞辱或貶低他人
- 在未經許可的情況下公開他人的隱私訊息
- 任何形式的騷擾（無論是公開或私下）
- 其他可能被合理認為在專業環境中不當的行為

## 維護者的責任

專案維護者有責任闡明此行為準則的標準，並將針對任何不當、威脅、冒犯或有害的行為採取適當且公平的糾正行動。

## 適用範圍

此行為準則適用於與專案相關的所有實體與虛擬空間，包括 GitHub Issue、Pull Request、討論區以及社交媒體。

## 檢舉流程

如果您遇到或目睹違反此行為準則的行為，請聯繫專案擁有者或維護者（例如透過專案的開發頻道）。所有舉報都將被保密並得到妥善處理。
